## Что добавляем

1. **Селектор адресата в форме создания поручения** — выпадающий список сотрудников из текущей компании (по умолчанию — сам пользователь).
2. **Привязка поручений к KR** — в карточке цели у каждого Key Result появляется кнопка «Поручения», открывающая диалог: список уже привязанных задач + селектор «привязать существующее поручение».

Бэкенд **не трогаем** — таблица `tracker_task_goal_links` уже поддерживает поле `key_result_id`, миграция и политики готовы.

## Файлы

### Новый: `src/components/tracker/EmployeePicker.tsx`
Лёгкий select поверх shadcn `Select`. Внутри:
- `useQuery` грузит `profiles (user_id, full_name)` через `laravelDb.from("profiles").select(...)` (бэкенд автоматически фильтрует по компании текущего пользователя через `CompanyScope`; суперадмин получит всех).
- Пропсы: `value`, `onChange`, `placeholder`, опционально `companyId` (на будущее).

### Изменяем: `src/pages/tracker/TrackerTasks.tsx`
В `TaskCreateDialog`:
- добавляем поле `assignee_id` в локальный стейт, дефолт = `useEffectiveUserId()`;
- рендерим `<EmployeePicker>` перед блоком «Срочность/Срок»;
- передаём `assignee_id` в `create.mutateAsync(...)`.
В `TaskRow` показываем имя адресата под заголовком (если есть в загруженных профилях — иначе скрываем).

### Изменяем: `src/hooks/tracker.ts`
- Новый хук `useKrTaskLinks(krId)` — `tracker_task_goal_links.select("*, task:tracker_tasks(id,title,status,urgency)").eq("key_result_id", krId)`.  
  Использует уже поддерживаемый `applySelect` (eager-load в `DbController`).
- Маленькое расширение `useUnlinkTaskFromGoal` → переименовать в `useUnlinkLink` (с обратной совместимостью) либо добавить `useUnlinkKrTask`, который инвалидирует ключи `tracker.krLinks`.

### Изменяем: `src/pages/tracker/TrackerGoals.tsx`
В `KeyResultsBlock`, рядом с кнопкой удаления KR, кнопка `<Link2 />` «Поручения (n)». Открывает Dialog:
- список привязанных задач (title + статус + кнопка «отвязать»);
- ниже — select «Добавить» из списка `useTasks()` той же компании, минус уже привязанные → `useLinkTaskToGoal({ task_id, goal_id: kr.goal_id, key_result_id: kr.id })`.

## Что НЕ делаем

- Не создаём новые поручения изнутри диалога KR (только привязка существующих) — оставим на следующую итерацию.
- Не правим политики, миграции, типы в БД.
- Не трогаем баг с `company_id` для суперадмина (про него уже знаем; это отдельный сюжет).

## Проверка

После изменений: создать поручение с указанием адресата ≠ себя; открыть OKR → раскрыть KR → привязать к нему созданное поручение → убедиться, что список обновляется и счётчик в кнопке увеличивается.
