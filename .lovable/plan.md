# Проверка дизайн-системы сайдбара — найденные ошибки

Тайпчек чистый, i18n-ключи на месте. Ниже — реальные логические/UX баги и предлагаемые правки.

## 1. Закрытая группа скрывает активный пункт (UX-баг)

В `AppSidebar.tsx` (строка 277):
```ts
const isOpen = collapsed ? false : (openGroups[entry.label] ?? hasActive);
```
Если пользователь вручную свернул группу (`openGroups[label] = false`), а затем перешёл по ссылке на её ребёнка — `??` не сработает (значение `false`, а не `undefined`), и активный пункт окажется скрыт.

**Фикс:** если `hasActive === true`, форсить раскрытие:
```ts
const isOpen = collapsed ? false : (hasActive || !!openGroups[entry.label]);
```

## 2. Возможные дубликаты React-ключей

В `renderEntry` ключ листового пункта — `item.path + item.label`. Один и тот же `tracker` (`/tracker/dashboard`, label «Трекер задач») рендерится сейчас только в одной секции на роль, но логика хрупкая — при добавлении того же пункта в две секции получим duplicate-key warning.

**Фикс:** включить в key идентификатор секции, например `${section.key}:${item.path}` (передать `section.key` в `renderEntry`).

## 3. На десктопе нельзя полностью спрятать сайдбар

В `AppLayout.tsx` `toggleSidebar` на десктопе только переключает `collapsed`. `onHide` пробрасывается в сайдбар, но кнопка-шеврон вызывает `onToggle`, а не `onHide`. То есть `PanelLeftOpen` в шапке появится только если `hidden=true`, но способа выставить `hidden=true` на десктопе нет.

**Фикс:** либо убрать ветку `hidden && <PanelLeftOpen/>` для десктопа как мёртвую, либо добавить отдельную кнопку «скрыть» в свернутом состоянии (Ctrl-клик по шеврону, или маленькая кнопка `X` сверху для десктопа). Минимальный безопасный вариант — оставить как есть и просто убрать неиспользуемый `onHide` для не-mobile, чтобы не вводить в заблуждение.

## 4. Фильтр пустых секций — мёртвый код

```ts
const sections = getSections().filter((s) => s.entries.length > 0);
```
В текущей реализации все ветки `getSections()` возвращают непустые `entries`. Если в будущем секция станет пустой из-за прав/флага — фильтр спасёт. Можно оставить, но стоит подтвердить, что это намеренно.

## 5. `useState(initialOpen)` инициализируется один раз

`initialOpen` пересчитывается на каждый рендер, но передаётся как initial value в `useState` — обновления `location.pathname` не подхватываются. После фикса №1 это перестаёт быть проблемой (fallback на `hasActive` всегда работает), отдельная правка не нужна.

## 6. Двойной `position: fixed` (информационно, не баг)

`AppSidebar` рендерится как `fixed` внутри `fixed`-обёртки в `AppLayout`. Работает за счёт `transform` на обёртке (transformed-ancestor превращает fixed в относительный к ней). Сейчас всё корректно — оставляем как есть, но желательно не убирать `transition-transform` с обёртки.

---

## Технический план правок (только в `src/components/AppSidebar.tsx`)

1. Поменять формулу `isOpen`:
   ```ts
   const isOpen = collapsed ? false : (hasActive || !!openGroups[entry.label]);
   ```
2. Передать `section.key` в `renderEntry(entry, sectionKey)` и собирать ключи как:
   - группа: `group:${sectionKey}:${entry.label}`
   - лист: `item:${sectionKey}:${item.path}`
3. (Опционально) Убрать ветку `hidden && <PanelLeftOpen/>` в `AppLayout.tsx` либо добавить десктоп-кнопку «полностью скрыть». Подтвердить желаемое поведение.

Бэкенд/бизнес-логику не трогаем — правки только в UI-слое сайдбара.
