## Цель
Переработать `AppSidebar` по принципу Photoshop-панелей: разбиение на смысловые блоки (уже есть), плюс новое поведение раскрытия — в развёрнутом виде блоки работают как аккордеоны-«свёртки», в свёрнутом — как иконки с hover-flyout, где раскрывается весь блок.

## Что меняется (только UI `src/components/AppSidebar.tsx`)

### 1. Блоки = единица сворачивания
Сейчас секции (`myWork`, `communication`, `analytics`, `hr`, `motivation`, `knowledge`, `system`) — это просто визуальные группы с лейблом. Делаем их полноценными «свёртками»:
- У каждой секции добавляется иконка-репрезентант (для compact-режима и для шапки свёртки): `Briefcase` / `MessageCircle` / `BarChart3` / `Users` / `Trophy` / `GraduationCap` / `Settings`.
- Внутри секции остаются обычные пункты и текущие подгруппы (`employeesGroup`, `careerGroup`, `performance`, `assessmentGroup`) — их поведение не меняется (вложенный второй уровень).

### 2. Развёрнутый режим (240px) — аккордеон
- Шапка секции = кликабельная кнопка `[иконка] [label] [chevron]`.
- По клику — toggle одной секции. Состояние хранится в `openSections: Record<string, boolean>`.
- По умолчанию **все секции свёрнуты**, кроме той, что содержит активный маршрут (как сейчас для групп).
- Можно держать открытыми несколько секций одновременно (как панели Photoshop).
- Состояние сохраняется в `localStorage` (`sidebar.openSections.v1`), чтобы не схлопывалось при навигации.

### 3. Свёрнутый режим (64px) — flyout по наведению
- В колонке отображается иконка каждой секции (вместо текущей россыпи отдельных иконок пунктов).
- На `mouseenter` иконки секции — справа от сайдбара появляется плавающий поп-овер (Radix `HoverCard` или `Popover` с `open` по hover) шириной ~240px, который рендерит ту же структуру, что и развёрнутая секция: лейбл + пункты + вложенные подгруппы.
- Закрытие на `mouseleave` иконки и панели (с маленьким delay, чтобы не схлопывалось при переходе курсора).
- Активный пункт внутри секции подсвечивает иконку секции (точка/полоса слева).

### 4. Шапка/лого/футер — без изменений
Trigger свёртки сайдбара, лого, плашка профиля, пункт выхода — остаются как есть.

## Технические детали
- Тип `NavSection` расширяется полем `icon: LucideIcon`.
- Новый компонент внутри файла: `SectionShell` — рендерит либо аккордеон (expanded), либо иконку + hover-flyout (collapsed).
- Для flyout используем уже доступный `@/components/ui/hover-card` (shadcn). Содержимое flyout переиспользует существующий `renderEntry`, поэтому подгруппы и активные состояния работают без дублирования логики.
- Активная секция вычисляется по `location.pathname` обходом `entries` (включая `children` групп).
- Совместимость с мобилкой: на `isMobile` flyout не применяется (сайдбар там всегда в drawer-режиме), используется аккордеон.

## Что НЕ меняется
- Состав пунктов и ролевая логика `getSections()`.
- Маршруты, переводы, i18n-ключи.
- Стили токенов, цвета, ширины сайдбара.
- Поведение вложенных групп второго уровня (`isGroup`).
