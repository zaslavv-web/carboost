## Часть 3. Профиль сотрудника `/me`

Единая личная страница, объединяющая то, что сейчас разбросано по Passport / UserProfileFull / Career Track. Файлы:

- Нов. `src/pages/MyProfile.tsx`
- Маршрут в `src/App.tsx`: `<Route path="/me" element={<MyProfile />} />`
- Локали `profile.*`

### Содержимое страницы (секции-карточки)

1. **Шапка**: аватар (с загрузкой), ФИО, должность, департамент, дата найма, контакт. Кнопка «Редактировать профиль».
2. **Чат-стикер / картинка для чатов**: отдельная загрузка картинки, которую сотрудник вставляет в свои сообщения. Превью + кнопки «Загрузить» / «Удалить».
3. **Текущий уровень и прогресс**: бейдж уровня, очки, прогресс-бар до следующего уровня, краткий список причин начисления (последние транзакции `currency_transactions`).
4. **Награды и достижения**:
  - Полученные награды (`employee_rewards` + `gamification_reward_types`) — сетка карточек с иконкой.
  - Личные достижения (`achievements`) — список.
5. **Карьерный трек**: компактный виджет текущего назначения (`employee_career_assignments` → этапы), кнопка «Открыть полностью» → `/career-track`.
6. **Цифровой паспорт**: сводка из Passport (skills, последние оценки) — ссылка «Полный паспорт» на `/passport` (страницу не удаляем, не дублируем содержимое глубоко).

### Загрузка изображений

- Аватар: уже есть `profiles.avatar_url` (если поле есть на бэке Laravel — используем существующий эндпоинт; иначе добавляем поле миграцией Laravel — фиксируем в подплане ниже).
- Чат-стикер: новое поле `profiles.chat_sticker_url`. Хранение в Laravel-хранилище через `laravelStorage`, по аналогии с `GamificationManagement`.
- Использование в чатах: в `src/components/chat/*` добавить кнопку «Вставить мою картинку» рядом с кнопкой эмодзи/вложений → подставляет markdown-картинку с URL из `profile.chat_sticker_url`. Это минимальная интеграция; полноценный медиа-пикер — отдельная задача.

> Поскольку слой данных — Laravel/Postgres (не Lovable Cloud), миграции/контроллеры на стороне бэкенда нужно будет описать как отдельный подпункт (нов. поле `chat_sticker_url`, маршрут upload). В рамках текущей итерации фронт делаю с фолбэком: если поле отсутствует — UI скрывается с подсказкой.

## Часть 4. Уровни в геймификации

Файлы: `src/pages/GamificationManagement.tsx` (новая вкладка «Уровни»), нов. `src/components/gamification/LevelBadge.tsx`, нов. хук `src/hooks/useEmployeeLevel.ts`. Бэкенд — Laravel-миграция.

### Модель уровней

Таблица `gamification_levels` (per company):

- `id`, `company_id`, `order` (1..N), `title` (например, «Новичок», «Профи», «Эксперт», «Магистр»), `icon`, `color`,
- `min_points` (порог по очкам),
- `min_tenure_months` (опционально, выслуга),
- `min_achievements_count` (опционально),
- `description`.

Логика расчёта (на сервере, RPC/эндпоинт `GET /gamification/levels/me`):

- Достижимый уровень = максимум по `order`, где все условия выполнены: `points ≥ min_points` И `tenure ≥ min_tenure_months` И `achievements_count ≥ min_achievements_count`.
- Прогресс до следующего уровня — по очкам (основной показатель).

### UI

**В `GamificationManagement.tsx**` — новая вкладка «Уровни»:

- Таблица уровней, drag-сортировка по `order`, CRUD-диалог (название, иконка, цвет, пороги).
- Превью «лестницы» уровней с подсветкой пользователей (счётчик «сотрудников на уровне X»).
- Дефолтные 5 уровней, создаются при первом открытии (seed).

**В `MyProfile.tsx**` — карточка «Уровень» (см. Часть 3.3).

**Список сотрудников** (`Employees.tsx`) и виджет признаний — рядом с очками показывать бейдж текущего уровня (компонент `LevelBadge`).

### Подплан Laravel-миграции (отдельно от текущей итерации)

```
- create gamification_levels (...)
- profiles.chat_sticker_url text null
- endpoint POST /me/chat-sticker (upload)
- endpoint GET /gamification/levels/me → { level, next, progress }
- endpoint GET/POST/PATCH/DELETE /gamification/levels
```

Без этих ручек фронт показывает уровни в режиме «mock из конфигурации компании», а загрузку стикера прячет — чтобы UI не падал.
