## Проблема

Сейчас `HRDEmployeeMap` рендерит всех 150+ сотрудников одним холстом ReactFlow: сплошные стрелки иерархии + пунктирные HR-задачи создают «клубок», текст мельче 8px, схема нечитаема.

## Решение — многоуровневая карта со ссылками на подсхемы (как в stormbpmn)

Три уровня детализации + хлебные крошки для быстрого возврата.

### Уровень 1. Обзор компании (Company Map)
- Ноды — **подразделения** (department). Внутри: название, число сотрудников, руководитель, средний балл, число активных HR-задач.
- Рёбра — связи «руководитель → команда» между руководителями департаментов (агрегированные `teamLinks`) и пунктирные HR-задачи между департаментами.
- Клик по ноде департамента → переход на уровень 2 (drill-in).
- Компактно, 10–30 нод максимум, читается на одном экране.

### Уровень 2. Департамент (Department Map)
- Ноды — **команды/руководители** этого департамента (группировка по `manager_id` или подотделу).
- Рёбра — иерархия внутри департамента + HR-задачи, где участник департамента.
- Клик по команде → уровень 3. Клик по «↑ вернуться» → уровень 1.

### Уровень 3. Команда (Team Map)
- Ноды — конкретные сотрудники команды (текущий вид, но не более 15–25 нод).
- Клик по сотруднику → правая панель с деталями (как сейчас).

### Навигация
- **Breadcrumbs** сверху: `Компания › Продукт › Команда Иванова`. Каждый сегмент — кликабельный, переносит на нужный уровень.
- **Кнопка «↑ Уровень выше»** слева.
- **Мини-карта** ReactFlow остаётся, но только для текущего уровня.
- Состояние уровня хранится в URL (`?level=dept&deptId=...&teamId=...`) — рефреш и «Назад» браузера работают корректно.
- Поиск сотрудника по имени: находит → сразу телепортирует на нужный уровень (team) с подсветкой ноды.

### Технически
Новые файлы:
- `src/components/hrd-map/CompanyLevel.tsx` — агрегирует департаменты, строит граф.
- `src/components/hrd-map/DepartmentLevel.tsx` — команды внутри департамента.
- `src/components/hrd-map/TeamLevel.tsx` — извлечённый текущий вид сотрудников (одной команды).
- `src/components/hrd-map/MapBreadcrumbs.tsx` — навигация уровней.
- `src/components/hrd-map/useMapNavigation.ts` — хук: URL-state + переходы.

Рефакторинг `HRDEmployeeMap.tsx`:
- Оставляет загрузку данных (employees, teamLinks, hrTasks, balances) и правую панель с деталями.
- Внутри `<Card>` вместо одного `<ReactFlow>` — `<MapBreadcrumbs />` + условный рендер уровня.
- Layout уровней: `dagre` для аккуратной иерархии (сверху вниз), вместо ручного `col*colW / idx*rowH` — сейчас команды визуально смешиваются.
- Пунктирные HR-задачи на уровне 1 и 2 агрегируются (одна линия = N задач между парой узлов, число выводится в label).

### Что не меняется
- Данные и запросы прежние, backend не трогаем.
- Правая панель сотрудника, HR-задачи, диалоги создания — как есть.
- Вкладки «Сотрудники / Карта / Треки / Заявки» и общий layout HRD-дашборда.

### Итог для пользователя
Верхний уровень читается сразу (10–30 крупных нод). Один клик — видно структуру департамента. Ещё клик — команда с деталями по сотруднику. Возврат — по breadcrumbs или кнопкой. Никаких «клубков» пунктиров через весь экран.
