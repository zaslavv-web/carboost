## План исправления

Проблема сейчас не в отправке почты как таковой: по коду `bulk_invite_employees` возвращает `created=0` и `mailed=0`, когда все строки попадают в `skipped` до этапа отправки. Самая вероятная подтвержденная по коду причина — существующее pending/claimed приглашение на этот email уже есть, а бэкенд молча пропускает такие строки без понятной ошибки и без повторной отправки.

### Что исправлю

1. **Бэкенд: сделать повторное приглашение идемпотентным**
  - Если email уже есть в `employee_invitations` со статусом `pending`, не возвращать «0/0».
  - Обновлять существующее pending-приглашение: ФИО, должность, отдел, роль, новый токен, `updated_at`.
  - Ставить это приглашение в очередь отправки письма.
  - В ответе вернуть отдельное поле вроде `resent`/`updated`, чтобы фронт мог показать: «обновлено и отправлено повторно».
2. **Бэкенд: не молчать по claimed/cancelled/дубликатам**
  - Для `claimed` возвращать понятную ошибку: пользователь уже принял приглашение.
  - Для некорректных email уже есть ошибка — оставлю.
  - Для skipped добавлю причину в `errors[]`, чтобы UI не показывал загадочное «приглашено 0, отправлено 0».
3. **Бэкенд: убрать хрупкую проверку несуществующего поля**
  - В self-healing ветке сейчас есть обращение к `$prof->department_id`, хотя по прошлой ошибке в `profiles` такого поля нет. В PHP это может давать warning/notice в зависимости от настроек.
  - Заменю на безопасную проверку через `property_exists` и основной путь через `position_id`.
4. **Фронт: нормальный тост вместо странного сообщения**
  - Если `created=0`, но есть `resent/updated > 0` и `mailed > 0`, показывать: «Приглашение отправлено повторно».
  - Если всё 0 — показывать первую конкретную причину из `errors[]`, а не сухие нули.
  - Если письмо не ушло — оставить предупреждение с предложением «Отправить повторно».
5. **Проверка**
  - Проверю, что ответ API для повторного pending email больше не превращается в `0/0`.
  - После деплоя на проде нужно будет снова выполнить `git pull`, `php artisan optimize:clear` и повторить приглашение того же тестового email.
  для очистки совести очисти очередь приглашений, которые были отправлены последние 2 часа
