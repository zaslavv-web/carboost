## Проблема

На скриншоте у HRD-дашборда заголовок «Панель HRD» и подзаголовок практически не читаются. Причина системная:

- Блок hero использует токен `.gradient-hero`, который **тёмный в обеих темах** (`hsl(220 11% 14%) → 22%` в light, и ещё темнее в dark — см. `src/index.css:85` и `:146`).
- Текст внутри hero написан как `text-foreground` / `text-muted-foreground`. В **светлой теме** `--foreground` почти чёрный → чёрный текст на тёмном фоне → нечитаемо.

Тот же паттерн используется в:

- `src/pages/HRDDashboard.tsx` (строки 452–472)
- `src/pages/Dashboard.tsx`
- `src/pages/Passport.tsx`
- `src/pages/MyProfile.tsx`
- `src/pages/Recognition.tsx`
- `src/pages/RiskAnalytics.tsx`
- `src/pages/Login.tsx`

То есть баг воспроизводится на всех hero-баннерах в светлой теме.

## Решение

Одно точечное изменение в `src/index.css` — переопределить локальные значения переменных `--foreground` и `--muted-foreground` внутри `.gradient-hero`, чтобы дочерний контент **автоматически** становился читаемым на тёмном градиенте. Никаких правок по семи страницам.

```css
.gradient-hero {
  background: var(--gradient-hero);
  /* Hero всегда тёмный — форсируем светлый контент */
  --foreground: 0 0% 100%;
  --muted-foreground: 0 0% 100% / 0.78;  /* fallback ниже на случай старых браузеров */
  color: hsl(var(--foreground));
}
.gradient-hero .text-muted-foreground { color: hsl(0 0% 100% / 0.78); }
.gradient-hero .text-foreground { color: hsl(0 0% 100%); }
```

Дополнительно подровняю две мелочи в hero HRD-дашборда (`HRDDashboard.tsx:465`):

- кнопка «Риски и удержание» (`variant="outline"` поверх `bg-background/40`) в светлой теме даёт тёмный текст на полупрозрачном тёмном фоне — добавлю явный светлый цвет, чтобы соответствовать.
