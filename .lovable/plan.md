## Проблема

`smtp:status` показывает 10 символов вместо 16. Значит парсер dotenv обрезает пароль. Типичные причины:

- **пробел** внутри значения без кавычек (Яндекс показывает пароль группами `xxxx xxxx xxxx xxxx` — если скопировать «как есть», dotenv возьмёт только до первого пробела);
- `**#**` в середине — всё после трактуется как комментарий;
- `**$**`, `**"**`, `**\**` без экранирования;
- две строки `MAIL_PASSWORD=` в файле — побеждает последняя (возможно, ниже остался старый огрызок длиной 10);
- BOM / неразрывный пробел `\xC2\xA0` при копировании из браузера.

В пароле только буквы латинского алфавита - никаких пробелов, спецсимволов и так далее

## План

### 1. Новая команда `php artisan smtp:env-doctor`

Файл: `backend-laravel/app/Console/Commands/SmtpEnvDoctor.php`

Что делает (без вывода самого пароля):

- читает `.env` напрямую через `file()` минуя dotenv-парсер;
- ищет **все** строки `MAIL_PASSWORD=` и `SMTP_PASSWORD=`, печатает номера строк и длину сырого значения;
- для каждой строки сообщает:
  - длину сырой части после `=`;
  - есть ли кавычки;
  - флаги наличия: пробел, `#`, `$`, `\`, неразрывный пробел `\xC2\xA0`, табуляция, CR;
  - длину «как увидел dotenv» (через `RuntimeEnv::get`);
- если строк больше одной — предупреждает про дубликат;
- проверяет BOM в начале файла;
- подсказывает корректный формат: `MAIL_PASSWORD="abcdefghijklmnop"`.

Сами значения паролей не печатаются — только длины и флаги.

### 2. Использование

```
php artisan smtp:env-doctor
```

По выводу сразу видно:

- найдено 2 строки → удалить лишнюю;
- `raw=16, parsed=10` + `has_space=true` → обернуть в двойные кавычки;
- `has_hash=true` → кавычки обязательны;
- `has_nbsp=true` → перенабрать пароль вручную.

После правки: `php artisan optimize:clear && php artisan smtp:status` → длина 16 → `php artisan smtp:test growthpeak@yandex.ru`.
