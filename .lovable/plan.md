# План: трекер уровня Jira + Trello-канбан

Сейчас в модуле «Трекер» есть Поручения (tasks), Цели/KR (goals/key_results), связи task↔KR и 1:1. Чтобы закрыть сценарии Jira (issue-tracker, scrum/kanban, workflow) и Trello (визуальный канбан с drag-and-drop), нужно достроить нижеперечисленное.

## 1. Анализ функциональности конкурентов

**Jira (что обязательно):**

- Проекты / пространства, разделение задач по проектам
- Типы задач (Epic, Story, Task, Bug, Sub-task) и иерархия parent/child
- Настраиваемые статусы и workflow (переходы между статусами)
- Спринты и беклог (Scrum), доска Kanban с WIP-лимитами
- Поля: priority, story points, estimate, time tracking, labels, components, fix version
- Комментарии, вложения, ментионы, активити-лог (история изменений)
- Watchers / подписки / уведомления
- Фильтры и сохранённые представления, поиск (JQL-аналог)
- Релизы / версии, roadmap
- Интеграции (репозитории, CI), webhooks

**Trello (что обязательно):**

- Доски с колонками-статусами и карточками
- Drag-and-drop между колонками и внутри колонки (порядок)
- Чек-листы внутри карточки, метки (цветные), сроки, обложки
- Множественные исполнители, активити на карточке
- Шаблоны досок и карточек

## 2. Что уже есть и что добавить


| Область           | Сейчас                  | Добавить                                                                           |
| ----------------- | ----------------------- | ---------------------------------------------------------------------------------- |
| Сущности          | tasks, goals, KR, links | projects, epics, sprints, comments, attachments, activity, watchers, labels        |
| Иерархия          | плоско                  | epic → story/task → sub-task, parent_id                                            |
| Статусы           | фикс. enum              | workflow per project (states + transitions)                                        |

## 3. Поэтапная разработка

### Этап 1 — Фундамент (проекты + расширенная задача)

- Таблица `tracker_projects` (company_id, key, name, lead_id, описание, иконка).
- Расширение `tracker_tasks`: `project_id`, `parent_id`, `type` (epic/story/task/bug/subtask), `priority`, `story_points`, `estimate_minutes`, `labels` (jsonb), `order_index`, `start_at`, `sprint_id` nullable.
- Селектор проекта в шапке трекера, фильтры по проекту во всех экранах.
- Страница «Проекты» (CRUD), роль project lead.

### Этап 2 — Канбан в стиле Trello

- Новая страница `TrackerBoard` для проекта: колонки = статусы workflow, карточки = задачи.
- Drag-and-drop через `@dnd-kit/core` (порядок сохраняется в `order_index`, смена колонки = смена статуса).
- Карточка: метки, исполнитель, срок, story points, чек-листы (mini).
- Быстрое создание задачи прямо в колонке.
- WIP-лимит на колонку (поле в workflow).

### Этап 3 — Workflow и типы задач

- Таблицы `tracker_workflows`, `tracker_workflow_states`, `tracker_workflow_transitions` (на проект).
- Замена жёсткого enum статусов на ссылку `state_id`.
- Иконки/цвета статусов, обязательные условия перехода (например, нельзя в Done без story_points).
- UI редактора workflow для админа проекта.

### Этап 4 — Спринты и беклог (Scrum)

- Таблица `tracker_sprints` (project_id, name, goal, start_at, end_at, status: planned/active/closed).
- Страница «Беклог»: список задач без спринта + перетаскивание в спринт.
- Страница «Активный спринт»: канбан, отфильтрованный по `sprint_id = active`.
- Burndown-чарт (Recharts) — остаток story points по дням.

### Этап 5 — Совместная работа

- `tracker_comments` (task_id, author_id, body markdown, created_at) + ответы.
- @упоминания сотрудников (EmployeePicker → подстановка), уведомление в шапке.
- `tracker_activity` лог (изменение поля → запись). Отображение на карточке.
- `tracker_watchers` (task_id, user_id). Автоподписка автора и исполнителя.
- Вложения: Lovable Cloud Storage bucket `tracker-attachments`, таблица `tracker_attachments`.

### Этап 6 — Поиск, фильтры, представления

- Глобальный поиск по проекту (текст + фильтры: assignee, label, type, priority, sprint, status, due).
- Сохранённые фильтры на пользователя.
- Дополнительные представления: список (как сейчас), календарь (по `due_at`), роадмап эпиков (Gantt-lite на React Flow или Recharts timeline).

### Этап 7 — Связи и иерархия

- Расширение `tracker_task_goal_links` → универсальные `tracker_task_links` (типы: blocks, is blocked by, relates to, duplicates, parent/child).
- Дерево подзадач на странице задачи.
- Привязка эпика к OKR/KR (уже частично есть).

### Этап 8 — Уведомления и интеграции

- Внутрипортальные уведомления (используем существующий механизм + новый тип `tracker_*`).
- E-mail дайджест по подпискам (через существующий бэкенд).
- Webhooks исходящие (для CI/Git), на будущее.

## 4. Технические детали

- **Бэкенд:** Laravel (`laravelDb` MySQL). Все новые таблицы получают `company_id`, скоупятся в `BasePolicy`. Миграции и контроллеры готовятся на стороне Laravel, фронт работает через существующий REST-слой `laravelDb.from(...)`.
- **Frontend:** TanStack Query (паттерн как в `src/hooks/tracker.ts`), Shadcn UI, теал/слейт токены. Канбан — `@dnd-kit/core` (нужно `bun add @dnd-kit/core @dnd-kit/sortable`).
- **Локализация ошибок:** русский (через существующий toast-обработчик).
- **Мультиарендность:** все запросы фильтруются по `company_id`; супер-админ видит всё.
- **Совместимость с уже сделанным:** существующие задачи получают `project_id = null` (виртуальный «Inbox»-проект на компанию), при первом открытии мастер предлагает выбрать проект.
- **Объём:** 8 этапов, каждый — отдельная итерация со своими миграциями и UI; рекомендую идти по порядку, чтобы продукт оставался работоспособным после каждого этапа.
