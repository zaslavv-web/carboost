## Проблема

На мобильной версии тап «Да, выйти» в `MobileEmployeeLayout` приводит к зависанию UI (белый/замороженный экран, невозможно тапать).

## Диагноз (по коду, не подтверждено рантайм-логами)

В `src/components/MobileEmployeeLayout.tsx:96` обработчик:

```ts
onClick={async () => { await signOut(); navigate("/login"); }}
```

делает три вещи, каждая из которых на мобилке способна «подвесить» интерфейс:

1. **AlertDialog не закрывается до навигации.** `logoutOpen` остаётся `true`, `signOut` идёт по сети (`POST /auth/logout` через Sanctum) — на слабом мобильном соединении это секунды, всё это время диалог блокирует UI. Плюс перед этим только что закрывался `Sheet` (строка 84 закрывает Sheet и сразу открывает AlertDialog) — у Radix при перекрытии двух порталов известная гонка, оставляющая `pointer-events: none` / `aria-hidden` на `<body>`, из-за чего после редиректа страница «мертвая».
2. `**navigate("/login")` через React Router не сбрасывает состояние Radix-порталов.** `ProtectedRoute` и так редиректит после `setUser(null)`, но оверлеи диалога/шита остаются в DOM с залоченным скроллом.
3. `**await signOut()` блокирует навигацию, если запрос падает по таймауту.** В `LaravelAuthContext` `signOut` уже обёрнут в try/finally, но пока промис висит — юзер видит замерший диалог.

Тот же паттерн есть в `AppSidebar.tsx:531`, но на десктопе меньше проявляется, потому что нет вложенного Sheet и focus-trap ведёт себя корректнее.

## Что чинить (только фронт, презентационный слой)

**Файл `src/components/MobileEmployeeLayout.tsx`:**

1. В обработчике «Да, выйти» сначала закрыть диалог (`setLogoutOpen(false)`), затем на следующем тике вызвать `signOut()` — не awaitить сетевой ответ для UI, редирект делать через `window.location.replace("/login")`, чтобы гарантированно сбросить состояние Radix-порталов и React-дерева.
2. Также сбросить `menuOpen` на всякий случай.

**Файл `src/components/AppSidebar.tsx` (строка 531):** применить ту же правку для единообразия — закрыть `logoutOpen`, `signOut` без await, `window.location.replace("/login")`.

Итоговый паттерн:

```ts
onClick={() => {
  setLogoutOpen(false);
  setMenuOpen(false);
  void signOut();               // fire-and-forget, logout эндпоинт не критичен
  window.location.replace("/login");
}}
```

## Что НЕ трогаем

- `LaravelAuthContext.signOut` и `clearSession` — они работают корректно, проблема чисто в UI-обёртке.
- Логику `ProtectedRoute` — редирект и так сработает, но `window.location.replace` даёт чистый маунт без остаточных порталов.
