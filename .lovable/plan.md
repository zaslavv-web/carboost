## Цель

Переделать `src/pages/Landing.tsx` в духе kontur.ru: **минимум текста, максимум визуала и движения**. Текущий лендинг — длинные блоки прозы и таблиц; нужен «продуктовый витринный» лендинг, где смысл считывается за 3 секунды глазами.

## Опорный референс (kontur.ru)

- Гигантская тайловая сетка продуктов как главный экран — каждая плитка = иконка/иллюстрация + 1–2 слова + микровзаимодействие при hover.
- Бренд-цвет агрессивно используется как заливка плиток, не как акцент.
- Текст работает заголовками-«плакатами», а не абзацами.
- Анимация постоянная, но сдержанная: parallax на hover, marquee логотипов, счётчики, плавное появление при скролле.

## Новая структура лендинга (6 экранов, без воды)

```
┌─────────────────────────────────────────────┐
│ 1. HERO — большой плакатный заголовок       │
│    + анимированная мозаика 16 модулей       │
├─────────────────────────────────────────────┤
│ 2. МОДУЛИ — Bento-сетка плиток              │
│    (одна плитка = один модуль, иконка+слово)│
│    hover → подсветка + микро-иллюстрация    │
├─────────────────────────────────────────────┤
│ 3. ДЛЯ КОГО — переключатель ролей           │
│    (HRD / Руководитель / Сотрудник)         │
│    под ним живой скриншот интерфейса        │
├─────────────────────────────────────────────┤
│ 4. ЦИФРЫ — 4 счётчика с count-up анимацией  │
│    (например: 5+ инструментов → 1)          │
├─────────────────────────────────────────────┤
│ 5. КЛИЕНТЫ — бесконечный marquee логотипов  │
├─────────────────────────────────────────────┤
│ 6. CTA — крупный финальный блок             │
│    «Внедрить» / «Демо»                      │
└─────────────────────────────────────────────┘
```

### Конкретика по экранам

**1. Hero.** Слева — заголовок в 2–3 коротких слова (пример: «HR. Один экран.»). Справа — анимированная мозаика 4×4 из иконок модулей: плитки плавно «дышат» (stagger fade-in + лёгкий float). Над фолдом — только заголовок, кнопка «Демо», и мозаика. Никаких абзацев.

**2. Модули.** Bento 4-колоночная сетка. Каждая плитка использует фирменный антикварный золотой как заливку на 1 из 8 ключевых плиток (LMS, Карьера, Цели, Признание, Найм, Адаптация, Аналитика, AI). Hover → плитка приподнимается, иконка вращается/масштабируется, появляется одна строка-подзаголовок. Текст на плитке ≤ 3 слов.

**3. Для кого.** Три таба-пилюли (HRD / Руководитель / Сотрудник). При переключении — кросс-фейд скриншота интерфейса (используем уже сгенерированные мокапы дашбордов из проекта или сгенерируем 3 PNG-мокапа). Подпись 1 строкой.

**4. Цифры.** 4 крупных числа, считаются от 0 при попадании в viewport. Подпись микро-шрифтом сверху. Пример: «1 продукт вместо 5», «16 модулей», «100% RU-инфраструктура», «2–8 недель внедрения».

**5. Логотипы.** Бесконечная горизонтальная лента (CSS keyframes, без библиотек). Серые SVG-плейсхолдеры — потом заменим на реальные.

**6. CTA.** Полноширинный блок с тёмным фоном, одной строкой заголовка и двумя кнопками. Без формы здесь — она открывается в модалке (уже есть `DemoRequestDialog`).

## Визуальный язык

- **Палитра**: оставляем тёмную базу (Charcoal `220 11% 9%`) + Antique Gold `46 65% 56%` как акцент. Хиро и CTA на тёмном, секции модулей/ролей на светлом сурфейсе.
- **Типографика**: уже подключён `Instrument Serif` (для плакатных заголовков) + `Inter` (для UI). Заголовки секций — serif, очень крупно (clamp 48–96px), без All-Caps.
- **Сетка**: общий контейнер `max-w-[1400px]`, плотные bento-блоки.
- **Иконки**: уже есть `lucide-react` — используем её для иконок плиток, без 3D-рендеров.

## Анимация (без новых зависимостей)

Не подключаем framer-motion/GSAP. Хватит того, что уже есть:

- Tailwind `animate-fade-in`, `animate-scale-in`, `hover-scale`, `hover-lift` — уже в проекте.
- `IntersectionObserver` (нативный) для триггера счётчиков и stagger-появления плиток — небольшой хук `useInView`.
- CSS `@keyframes float`, `marquee` — добавим в `src/index.css`.
- `transform/opacity` на hover для плиток — чистый CSS.

Это держит бандл лёгким и не тянет ещё одну либу.
