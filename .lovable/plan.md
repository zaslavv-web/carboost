## Объём

Согласовано: правки идут в один поток, в порядке приоритета — сначала визуал, потом UX, потом функционал. После каждого блока — проверка билда/типов и быстрая прогонка ключевых страниц.

---

## Блок 1. Визуал и типографика (приоритет №1)

1. **Палитра** — переключаем на «Paper & Ink» (чистый off-white `#F5F3EE` + ink `#0D0D0D`) с одним сдержанным акцентом (текущий teal оставляем как accent, без неоновых значений). Все цвета — только через токены `index.css` + `tailwind.config.ts`. Уберём «грязные» серо-зелёные подложки.
2. **Шрифты** — заголовки `Instrument Serif` (или `DM Serif Display`), тело `Inter` (оставляем). Подключаем через `@fontsource/*`. Убираем All-Caps у длинных заголовков (`uppercase` → `normal-case`), оставляем только у бейджей/чипов до 3 слов.
3. **Иконки/иллюстрации** — заменяем устаревшие декоративные SVG/эмодзи на современные 3D-иллюстрации (генерация `imagegen` premium, прозрачный фон) для пустых состояний, hero и онбординга. Lucide остаются для UI-кнопок.
4. **Подложки** — ревизия `Card`/`bg-muted`/`bg-card` оболочек: вложенные подложки убираем, оставляем один уровень фонового контейнера; для разделителей — `border` или `space-y`.
5. **Микро-типографика** — non-breaking spaces для предлогов (`и`, `до`, `в`, `на`, `с`) через утилиту `nbspify(text)`, прогоняем заголовки/подписи лендинга и дашборда.
6. проверяем чтобы расстояние между строками было не меньше 2-3 пикселей
7. проверяем чтобы не было висячих предлогов - если строка заканчивается предлогом, его нужно перенести на следующую строку
8. проверяем чтобы не было переносов слов - лучше уменьшить шрифт

## Блок 2. Реструктуризация дашборда

- **Удаляем верхнюю панель** (`Topbar` с поиском/языком/именем/уведомлениями).
- В `AppSidebar`:
  - сверху — поиск (cmd+k);
  - в футере сайдбара — профиль (аватар + меню), уведомления (значок с поповером), язык (в подменю профиля).
- `SidebarTrigger` остаётся как маленькая кнопка в углу контента (для свёрнутого режима).
- Контент получает всю вертикаль; убираем фиксированную высоту хедера.

## Блок 3. UX-чистка компонентов

- «Сотрудники / Карты сотрудников» — переделать в табы (`Tabs` shadcn), а не кнопки.
- Объединить `Все сотрудники` и `Управление пользователями` в одну страницу с табами `Список / Роли и доступы`.
- Карточки: убрать кнопку «Подробнее», весь блок кликабелен, в правом верхнем углу — иконка `ArrowUpRight` (стрелка 45°).
- Раздел «Награды» при пустом состоянии скрываем (рендерим только если `rewards.length > 0`); иначе мини-плашка «Получите первую награду →» в профиле.
- Чек-лист онбординга «Компания» → переносим из пользовательского меню в `MyProfile` (раздел «Первичная настройка») + первый запуск показывает мастер.

## Блок 4. AI-настройки

- Прячем выбор модели/ввод API-ключа от обычного пользователя.
- В Settings оставляем переключатель «Расширенный режим» (по умолчанию выкл). Включённый — открывает текущий блок (модель, ключ).
- Глобальная модель и ключ управляются только Superadmin (раздел Superadmin → «AI-провайдер»). По умолчанию — серверная модель Lovable AI Gateway.
- На аналитических виджетах (`Средний балл по отделам`, риски, вовлечённость) — иконка `Sparkles`/`Info`, по клику — поповер с кратким объяснением методики (генерируем через AI с кэшем по `metric_id`).
- ко всем ai-компонентам добавляем тултипы с объяснением как считается

## Блок 5. Канбан без обязательного проекта

- Личные задачи: разрешаем создание без `project_id` (NULL допустим, миграция + политика).
- Колонки по умолчанию: `Backlog`, `In Progress`, `Testing`, `Done`. Сохранить кастомные workflow для проектов.
- История передач: новая таблица `task_transitions (task_id, from_user, to_user, from_status, to_status, at, by)`. Лента на правой панели задачи.

## Блок 6. Telegram-бот (ежедневный дайджест)

- Подключаем Telegram-коннектор (`standard_connectors--connect`).
- В профиле — поле «Telegram chat_id» + кнопка «Привязать» (отправляем код, бот валидирует через webhook).
- Edge Function `telegram-daily-digest` + cron 09:00 локального TZ пользователя: отправляет список задач на сегодня + просроченные + ближайшие дедлайны.
- Webhook `telegram-webhook` для команд `/today`, `/overdue`, `/link <code>`.

## Блок 7. Чат-виджет

- Полностью убираем плавающую кнопку в правом нижнем углу с дашборда (внутренний поиск/мессенджер переезжает внутрь сайдбара как пункт «Сообщения»).

## Блок 8. LMS (Университет)

- Оставляем модуль, доводим: миграция на нормальную структуру `course → modules → lessons`, поддержка вложений (видео — embed YouTube/VK/Rutube вместо собственного хостинга на этом этапе), markdown-редактор (`react-markdown` + `@uiw/react-md-editor`), прогресс по урокам.
- Этот блок — самый крупный, разносим на финал; если по времени не успеваем — фиксируем как отдельный следующий шаг.

## Блок 9. Лендинг

- Перепишем hero и 3 ключевых блока под выгоды: удержание, эффективность, экономия.
- Уберём «стены текста», заменим на скриншоты UI (используем готовые / новые премиум-изображения).
- Прогоняем nbspify, поправим All-Caps.

## Технические детали

- Все цвета — через CSS-переменные в `src/index.css` (HSL) + `tailwind.config.ts`. Запрещены `text-white`, `bg-black`, `bg-[#…]` в компонентах.
- Шрифты — `bun add @fontsource/instrument-serif @fontsource/inter`, импорт в `src/main.tsx`.
- 3D-иллюстрации генерируем `imagegen` (premium, `transparent_background: true`) в `src/assets/3d/`, заливаем через `lovable-assets`.
- Утилита `nbspify` — `src/lib/typography.ts`.
- Канбан: миграция Laravel — `tasks.project_id nullable`, новая таблица `task_transitions`, observer пишет переходы.
- Telegram: коннектор + 2 Edge Functions, cron через pg_cron (или Laravel scheduler, если бот живёт на стороне Laravel — уточнить по факту реализации).
- AI-подсказки: новая таблица `metric_explanations (metric_id, locale, body, updated_at)` + edge function `explain-metric`.

## Порядок и проверки

1. Блоки 1–3 (визуал + дашборд + чистка) — самые широкие, делаем первыми, между ними прогон `tsgo` + быстрый визуальный smoke (Playwright скрин на 3 ключевых страницах).
2. Блок 4 (AI-настройки) + блок 7 (чат-виджет) — короткие.
3. Блок 5 (канбан) + блок 6 (Telegram) — отдельный заход с миграциями.
4. Блок 8 (LMS) и блок 9 (лендинг) — финал.

## Что НЕ входит

- Полная переделка LMS-плеера/хостинга видео — только embed-источники.
- Кастомные Telegram-сценарии сверх дайджеста и `/today`, `/overdue`, `/link`.
- Замена ИИ-провайдера по умолчанию (остаётся текущий Gemini).
