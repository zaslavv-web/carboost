
## Идея

Сейчас все модули продукта — это «страницы с табами и карточками по центру». Слишком много воздуха, контекст задачи теряется при кликах, инструменты не под рукой. Figma и Photoshop решают это одинаково: **узкий рельс инструментов слева → большая рабочая канва в центре → правый стек инспекторов с настройками выбранного объекта → статус-бар внизу**. Берём именно эту модель.

Плотность — 4/5 (плотно, но без «фотошопного» микро-шрифта). Базовый шаг сетки 4px, типографика 12–13px в чипах/полях, 14px в основном тексте.

## Каркас (общий для всех «рабочих» модулей)

```text
┌──────────────────────────────────────────────────────────────────────────┐
│  Глобальный AppSidebar (240/64 px, как сейчас)                            │
├──┬──────────────┬──────────────────────────────────┬─────────────────────┤
│  │ ContextRail  │ Canvas (доска / графики / трек) │ Inspector Stack     │
│  │  44 px       │ flex-1                           │ 320 px (dock)       │
│  │  иконки      │ ─ Toolbar 36 px ───────────────  │ ── секции свора-    │
│  │  + tooltip   │ ─ Контент во всю ширину/высоту   │     чиваемые        │
│  │              │                                  │                     │
│  ├──────────────┴──────────────────────────────────┴─────────────────────┤
│  │ StatusBar 28 px: счётчики, фильтры-чипы, hotkeys, zoom, presence       │
│  └────────────────────────────────────────────────────────────────────────┘
```

Новые общие компоненты:

- `src/components/workspace/WorkspaceShell.tsx` — флекс-каркас (`ContextRail` + `Canvas` + `InspectorStack` + `StatusBar`).
- `src/components/workspace/ContextRail.tsx` — вертикальный рельс 44px, иконки-инструменты с тултипами и активным состоянием (как тулбар PS).
- `src/components/workspace/InspectorStack.tsx` — правая колонка 320px, ширина настраивается drag-резайзером, можно полностью свернуть в полоску 28px (как палитры PS). Содержит `InspectorSection` (свёртываемые блоки).
- `src/components/workspace/CanvasToolbar.tsx` — горизонтальная панель действий над канвой (фильтры, поиск, переключатели вида).
- `src/components/workspace/StatusBar.tsx` — нижняя строка: счётчики выделения, активный проект/период, hotkeys, индикатор сохранения.
- Ширины/состояния докинга хранятся в `localStorage` по ключу `workspace:<module>:<panel>`, чтобы каждый юзер собрал «своё».

Плюс утилиты: `useDockPanel(key, defaultWidth)`, `useHotkeys` (через существующий стек или мини-обёртка над `keydown`).

## Применение по модулям

### 1) Трекер (Доска / Бэклог / Задачи)

`src/pages/tracker/TrackerLayout.tsx` переписываем под `WorkspaceShell`. Вместо широких табов сверху — компактный **сегмент-переключатель видов** в `CanvasToolbar` (Дашборд · Доска · Бэклог · Цели · 1:1 · Workflows). Селектор проекта переезжает в `ContextRail` сверху (как «файл» в Figma) — клик открывает popover со списком и поиском.

- `ContextRail`: иконки видов (Board, Backlog, Goals, 1:1, Workflows, Reports). Активная иконка подсвечена primary-полоской.
- `CanvasToolbar` (доска): поиск, фильтры (assignee/label/sprint), переключатель Kanban/Swimlanes, кнопки «Новый эпик/история».
- Доска во всю ширину канвы, колонки автоматически растягиваются. Карточки уплотняем: один ряд высоты 28px для заголовка, чипы 18px.
- `InspectorStack` (новый `TaskInspectorPanel`): открывается **в правой панели**, а не модалкой, при клике на задачу. Секции: Status & Assignee · Estimate & Sprint · Описание · Чек-листы · Связи (epic/story) · Активность. Каждая секция — отдельный `InspectorSection`, сворачивается. Закрытие — крестик в шапке секции «Задача».
- `StatusBar`: «12 выбрано · Sprint 24 · 3 фильтра активны · ⌘K — команды».
- Бэклог: вместо стопки карточек — таблица + правый инспектор той же истории при выделении. То же для «Поручений» и «Целей (OKR)» — единый паттерн «список слева, инспектор справа».

Файлы:
- `src/pages/tracker/TrackerLayout.tsx` — переход на `WorkspaceShell`, переключатель видов вынесен в рельс.
- `src/pages/tracker/TrackerBoard.tsx`, `TrackerBacklog.tsx`, `TrackerTasks.tsx`, `TrackerGoals.tsx` — отдают `canvas` и `inspector` слоты в шелл; модалку `TaskDetailDialog` заменяем на `TaskInspectorPanel` (модалку оставляем как fallback на мобильных <md).

### 2) Career Tracks / Performance (визуальный редактор)

Это именно «канва + объект + свойства» — идеальный кейс PS-модели.

- `src/pages/CareerTracksManagement.tsx` и `CareerTrack.tsx`: рельс слева — инструменты (Курсор, Добавить шаг, Соединить, Группа, Комментарий), канва центром (существующий React Flow / Sankey), `InspectorStack` справа — свойства выделенного шага (название, длительность, требования, награды, AI-теги).
- В статус-баре: «3 шага выбрано · масштаб 80% · автосохранено в 14:32».
- `Performance.tsx`: левый рельс — фильтры периода/команды, центр — графики, правый инспектор — детали выбранного сотрудника/метрики (как сейчас `HRDEmployeeMap` детальная панель, но в общем каркасе).

### 3) HRD Dashboard и аналитика

`src/pages/HRDDashboard.tsx`, `Analytics.tsx`, `RiskAnalytics.tsx`:

- `ContextRail`: иконки разделов (Обзор, Риски, Текучка, Карьерные треки, Вовлечённость, Отчёты).
- `CanvasToolbar`: период, сегмент (компания/филиал/команда), сравнение «vs прошлый период», экспорт.
- Графики разворачиваются на всю ширину канвы, без лишних карточек-обёрток.
- `InspectorStack`: «Контекст метрики» — описание показателя, формула, бенчмарк, связанные метрики, кнопка «Создать алерт». Появляется при клике на любой график/число.

## Технические детали

1. **Резайз и докинг**: реализуем через `useDockPanel` + CSS grid (`grid-template-columns: 44px 1fr <inspectorWidth>`). Drag-handle — `<div role="separator">` 4px между канвой и инспектором, `cursor-col-resize`, обновляет ширину в state + localStorage.
2. **Сворачивание инспектора**: кнопка `PanelRightClose/Open` в шапке инспектора; в свёрнутом состоянии остаётся 28px вертикальная полоска с иконками открытых секций (как «свёрнутые палитры» в PS).
3. **Хоткеи**: `[` / `]` — свернуть/развернуть инспектор, `Tab` — спрятать обе боковые панели (фокус-режим), `⌘K` — командная палитра (отдельный шаг, в этот плана только хук).
4. **Адаптив**: на ширине <1024px правый инспектор автоматически становится `Sheet` (выезжает поверх). На <768px рельс схлопывается в горизонтальный bar над контентом.
5. **Темизация**: всё через существующие semantic-токены (`--background`, `--border`, `--muted`). Никаких хардкод-цветов. Рельс и инспектор получают `bg-card`, разделители — `border-border/60`.
6. **i18n**: новые строки в `common.json` под ключом `workspace.*` (RU/EN).
7. **Сохранность поведения**: бэкенд, политики, RPC и данные — без изменений. Это чисто UI-перекомпоновка.

## Что не трогаем

- Лендинг, авторизация, профили, чаты, магазин, гамификация — оставляем как есть.
- Мобильный layout сотрудника (`MobileEmployeeLayout`) — не меняется.
- Глобальный `AppSidebar` (он уже компактный) — без изменений, рельс модуля живёт правее него.

---

## Критерий приёмки

- В Трекере, Career Tracks/Performance и HRD-аналитике используется единый каркас `WorkspaceShell` с рельсом, инспектором и статус-баром.
- Клик по объекту (задача, шаг трека, метрика) открывает данные в правом инспекторе, а не в модалке.
- Инспектор резайзится, сворачивается, ширины запоминаются между сессиями.
- На 1440×900 канва занимает ≥ 60% ширины во всех трёх модулях; нет «пустых полей» по бокам.
- Все цвета — токены, тёмная тема не ломается.
