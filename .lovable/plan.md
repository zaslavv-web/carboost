## План доработок презентации `/investor-deck`

### 1. Навигация (переключатель слайдов)

Сейчас `DeckNav` — фиксированная плашка внизу по центру, наезжает на нижние блоки слайдов (особенно 5-й — «Точка безубыточности» и футер).

- Переносим `DeckNav` в **правый нижний угол** и делаем **вертикально-компактным**: круглые кнопки ▲/▼ (или ◀/▶) + счётчик, полупрозрачный фон.
- Добавляем **автоскрытие** при бездействии мыши >2.5 c (появляется при движении мыши, тапе на мобилке, нажатии клавиш). Печать/фуллскрин это уже учитывают.
- На узких экранах (< 900 px) прижимаем к правому краю по вертикали посередине.
- Клавиатурная навигация (◀ ▶ Space PageUp/Down F) остаётся.

### 2. Редактируемые тексты

Делаем **все текстовые узлы слайдов редактируемыми inline** с сохранением в `localStorage` (ключ `investor-deck:v1`).

- Единый компонент `<Editable id="slide5.team.role.0" as="span" defaultValue="..."/>` — `contentEditable` при клике, blur → сохранение. Поддерживает `multiline` (для абзацев) и `numeric` (для чисел в калькуляторе).
- Провайдер `DeckContentProvider` держит map `{ id → value }` и синхронизирует с `localStorage`. Дефолты берутся из текущих строк слайдов.
- В верхней панели `DeckNav` добавляем маленький toggle «Режим редактирования» + кнопку «Сбросить правки». Вне режима редактирования тексты — обычный DOM (без outline/hover-подсветки), чтобы презентация выглядела как обычно.
- Пробегаемся по всем 6 слайдам и оборачиваем текстовые литералы в `<Editable>` с уникальными `id`. Изображения/иконки/цифры графиков (Recharts data) не трогаем — только видимый пользовательский текст (заголовки, подзаголовки, буллеты, названия строк таблиц, подписи).

### 3. Анимации переходов

Сейчас `AnimatePresence` даёт только `opacity` fade за 0.35 c. Усиливаем:

- **Направленный slide+fade**: при переходе вперёд новый слайд въезжает справа (`x: 40 → 0`, `opacity 0 → 1`), старый уезжает влево. Назад — зеркально. Направление определяется по разнице индексов.
- Duration 0.45 c, easing `[0.22, 1, 0.36, 1]`.
- Внутри слайда — мягкий `stagger` на карточках (`children` fade+`y:12→0`, задержка 40 мс), реализуем через `motion.div` в `SlideLayout` вокруг `children`, чтобы правки были в одном месте.
- Уважаем `prefers-reduced-motion`: только opacity.

### 4. Размер шрифта: 10 – 60 px

Аудит и корректировка Tailwind arbitrary sizes во всех слайдах.

- **Минимум 10 px**: сейчас встречаются `text-[11px]` (kicker'ы, подписи) — поднимаем до `text-[12px]` (запас, комфортнее чем ровно 10 — но user требует «минимум 10», а не «ровно 10», значит всё что <10 надо поднять; 11 уже проходит). Проверю каждый файл — если найду `<10px`, подниму до 10.
- **Максимум 60 px**: `text-[64px]` (итог бюджета, hero-цифры) и `text-[58px]` (заголовки слайдов) — режем до `text-[60px]`. Также серифные `text-[68px]`/`text-[72px]` в других слайдах — ограничиваем 60.
- Прохожу по всем 6 слайдам + `SlideLayout` + `DeckNav`.

### 5. Калькулятор на слайде «Бюджет» (Slide5Economics)

Делаем ФОТ и маркетинг **редактируемыми числами**, всё остальное пересчитывается автоматически.

**Редактируемо (numeric inputs с валидацией ≥0):**

- В таблице «Команда · ФОТ» — колонка `monthly` для каждой роли (тыс. ₽/мес).
- В таблице «Продвижение · 12 мес» — колонка `v` для каждой статьи (тыс. ₽/год).
- Строка «Инфра / AI-тарифы» (`infraYear`) — тоже редактируемо (это по сути расходы, логично включить).
- Кол-во клиентов Y1/Y2/Y3, seat price, avg seats, grossMargin — оставляем как есть (не просили), НО выносим в состояние, чтобы в будущем легко расширить.

**Автопересчёт (реактивно через `useMemo`):**

- `teamMonthly`, `teamYear`, `promoYear`, `total`
- CAC = promoYear / clientsY1
- Payback = ceil(CAC / (ARPU × margin / 12))
- LTV, LTV/CAC
- ARR Y1/Y2/Y3
- `breakEvenClients` = ceil(total / (ARPU × margin))
- Донат-чарт (recharts) — данные из тех же состояний.

**UI-детали:**

- Числа в таблицах становятся `<NumericEditable>` — при клике превращаются в `<input type="number">` шириной по контенту, blur сохраняет. Стили — те же `font-mono text-right`, чтобы не ломать вёрстку. Правки сохраняются в тот же `localStorage`-стор (см. п.2), ключи вида `slide5.team.0.monthly`.
- Кнопка «Сбросить к дефолту» — рядом с заголовком карточки бюджета.

### Технические детали

**Новые файлы**

```
src/pages/investor/deck/DeckContentContext.tsx   — provider + hook useDeckValue(id, default)
src/pages/investor/deck/Editable.tsx              — <Editable> для текста
src/pages/investor/deck/NumericEditable.tsx       — <NumericEditable> для чисел
```

**Изменяемые файлы**

- `src/pages/investor/InvestorDeck.tsx` — оборачиваем в `DeckContentProvider`, добавляем direction в `AnimatePresence`, toggle «Edit mode».
- `src/pages/investor/DeckNav.tsx` — перенос в правый нижний угол, вертикальная компоновка, auto-hide, toggle edit.
- `src/pages/investor/SlideLayout.tsx` — оборачиваем `children` в `motion.div` со stagger, ограничиваем max font.
- `src/pages/investor/slides/Slide1..Slide6.tsx` — заменяем текстовые литералы на `<Editable>`, чиним размеры шрифтов (10–60 px). В `Slide5Economics` — полный рефактор в `useMemo` + `NumericEditable`.

**Что НЕ меняем**

- Данные Recharts-графиков (кроме донат-бюджета на слайде 5, который завязан на калькулятор).
- Логику печати (`@media print`), фуллскрин, keyboard shortcuts.
- URL-схему `?slide=N`.
- Дизайн-токены (цвета, шрифты).
