# Правки по разбору ЛК growth-peak.pro

Сгруппировал замечания в три волны: сначала быстрые UX-фиксы, потом технические баги, потом мобильная адаптация и тултипы. Внутри каждой волны правки независимы и могут идти параллельно.

## Волна 1. Дизайн / UX (быстрые правки)

1. **Красная точка на колокольчике** — показывать только при `unreadCount > 0` (NotificationsBell).
2. **Светлая тема — сайдбар не перекрашивается** — починить токены `--sidebar-*` для `:root` (сейчас стиль остаётся тёмным), а также цвет скроллбара сайдбара.
3. **Меню в светлой теме: активный пункт теряется** — добавить цвет текста для активного пункта в светлой теме (зеркально к недавнему `dark:text-white`).
4. **Тёмные «островки» в светлой теме** — лента признания и пр.: заменить хардкод `bg-slate-*` на семантические токены `bg-card/bg-muted`.
5. **Поля форм** — увеличить отступ между лейблом и инпутом (`space-y-2` минимум) в `Input`, `Textarea`, конфиге воркфлоу.
6. **Конфиг статусов воркфлоу** — упростить визуально: группировать «название / переходы / цвет» в карточки с разделителями, убрать визуальный шум.
7. **Иконка «связи» между названием и описанием в «Распределение ролей»** — заменить на нейтральный разделитель/убрать.
8. **Toasts (sonner)** — переместить из центра в правый нижний угол с auto-dismiss 4с, добавить кнопку закрытия.
9. **Logout** — добавить `AlertDialog` подтверждения «Выйти из системы? Да/Нет».
10. **Кнопка «Назад»** — увеличить размер (`size="default"` + иконка 20px), добавить hover-фон.

## Волна 2. Технические баги

11. **Открытие задачи → 500** — посмотреть `TaskController@show` / роут на бэке, починить сериализацию (вероятно нет загрузки relations).
12. **Загрузка файла (jpg) в задачу не работает** — проверить mime-фильтр и storage policy для `task-attachments`.
13. **«Создать курс» в Университете не работает** — починить onClick/роут.
14. **Раскрытие «закрытых» меняет счётчик активных** — общий counter, должен считать только активные.
15. **Дубль строки сотрудника при выделении** — исправить рендер списка (key + дедуп) и не выделять обе строки.
16. **1:1 — можно запланировать в прошлое** — добавить валидацию `min={now}` на дата-пикере и на бэке.
17. **Лого не грузится в трекере** — проверить путь логотипа компании (fallback на дефолтный лого).
18. **«Фирменный стиль»: ошибка при загрузке логотипа** — проверить эндпоинт `branding/logo` и storage permissions; вернуть локализованную ошибку.
19. **Цвет кнопок не сохраняется** — починить save endpoint `branding/colors`.
20. **Загрузка собственной аватарки — «недостаточно прав»** — в `ProfileController@updateAvatar` снять role-check, разрешить владельцу профиля.

## Волна 3. Мобайл / онбординг

21. **Таблицы дашборда на мобильном** — закрепить первый столбец (`sticky left-0`), сократить ширину, добавить пагинацию + выбор размера страницы (25/50/100).
22. **Тултипы** — добавить `Tooltip` на иконки/кнопки без подписи (приоритет: трекер, политики, оргструктура).
23. **Локализация** — пройти RU/EN, найти английские строки в русском режиме (особенно трекер, политики).
24. исправить положение иконки чата - сейчас он перекрывает нижнее меню

## Технические детали

- Все цвета — через токены `index.css` (никаких `text-white`/`bg-slate-900` в компонентах).
- Toasts: в `App.tsx` `<Toaster position="bottom-right" closeButton richColors />`.
- Logout-confirm: обернуть в `<AlertDialog>`, использовать существующий `useAuth().logout`.
- Аватарка: бэкенд — `ProfileController.php` → метод `updateAvatar`, убрать проверку роли, оставить проверку `auth()->id() === $user->id`.
- Branding: расследовать `BrandingController` и storage bucket `branding` (политики public read / owner write).
- 1:1: и фронт (`MeetingForm`), и бэк (`OneOnOneController@store`) — `scheduled_at >= now`.
- Сайдбар светлая тема: в `src/index.css` под `:root` задать `--sidebar-background`, `--sidebar-foreground`, `--sidebar-accent`, `--sidebar-accent-foreground`, `--sidebar-border` и `scrollbar-color`.
