
# Редизайн бокового меню: 5 блоков + дизайн в стиле биг-тех

## 1. Исследование: как сделано в крупных компаниях

Корпоративные порталы, которые я изучил/опираюсь как на референсы:

| Портал | Что хорошего | Что плохого |
|---|---|---|
| **Google Moma / Workspace** | Жёсткая группировка по доменам (People, Tools, Knowledge), section-label заглавными микро-капсом, иконки 20px. Боковое меню узкое (~240px), пункты сгруппированы 4-6 в раздел. | Группы статичны, не сворачиваются — у нас ролей много, нужна сворачиваемость. |
| **Amazon A-to-Z (Inside)** | Большие «карточки-плитки» наверху для топ-задач (Pay, Time, Benefits), остальное — текстовый список внизу. Меньше когнитивной нагрузки. | Плохо масштабируется на 30+ пунктов: часть прячется в «More». |
| **Microsoft MyHub / Viva** | Pinned-секция вверху (личное: задачи, сообщения), затем тематические аккордеоны. Цветные индикаторы активной секции слева (4px bar). | Аккордеоны иногда «прыгают», нужно фиксировать высоту/анимацию. |
| **Meta Workplace** | Двух-уровневая навигация: верхний рейл с крупными иконками-разделами + контекстная панель. Очень масштабируемо. | Дорого в реализации, требует переделки AppLayout — для нас избыточно сейчас. |
| **Workday / SAP SuccessFactors** | Section labels + сворачиваемые группы, бейджи-счётчики справа, hover показывает описание. Понятный паттерн для HR-домена. | Визуально «скучно», но это норма для HR. |

**Вывод:** берём гибрид Google + Microsoft Viva: 5 секций с заголовками-капсом, сворачиваемые группы (как у нас сейчас), активный пункт с левым 3px-баром (вместо полной заливки — меньше «грязи»), компактные 4px вертикальных промежутков, иконки 18px, ширина 240px (сейчас 260).

## 2. Пять блоков и распределение пунктов

| Блок | Иконка | Что входит |
|---|---|---|
| **Коммуникация** | MessageCircle | Чаты, Сообщения, Уведомления, Поддержка (тикеты), Признание (Recognition feed), Приглашения, Email-сервис (для SA) |
| **Аналитика** | BarChart3 | Дашборд, Аналитика, Risk Analytics, Product Analytics, Career Reviews, Сценарии/Тесты (как инструмент оценки) |
| **Управление персоналом** | Users | Сотрудники, Пользователи, Команда, Онбординг, Должности, Карьерные треки, Performance, Probation, Disciplinary, Leaves, HR-политики, Компании (SA), Pricing Inquiries (SA) |
| **Мотивация** | Trophy | Геймификация, Магазин/Shop Admin, Признание (если не дублировать в Коммуникацию), Карьерный трек (для Employee), AI-ассессмент (для Employee), Анкета сотрудника |
| **Полезные материалы** | BookText | Университет, RAG База знаний, Цифровой паспорт, Трекер задач |

Отдельно (вне блоков, внизу как «Системное»):
- Фирменный стиль, AI-провайдер, Настройки, Выход.

> Распределение строится per-role: у Employee часть блоков короче (например «Управление персоналом» = только Team у Manager / пусто у Employee, тогда блок не рендерится).

## 3. Визуальные изменения дизайн-системы

- Ширина: collapsed 64px / expanded 240px (было 72/260).
- Заголовок блока: `text-[10px] uppercase tracking-[0.14em] text-sidebar-foreground/40 px-3 pt-4 pb-1`. В collapsed — заменяется на тонкий разделитель `<div className="mx-3 my-2 h-px bg-sidebar-border/60" />`.
- Пункт: вертикальный padding `py-2` (было `py-2.5`), иконки `w-[18px] h-[18px]`.
- Активный пункт: убираем сплошную заливку — слева 3px бар `bg-sidebar-primary`, фон `bg-sidebar-primary/10`, текст `text-sidebar-primary-foreground`. Hover — `bg-sidebar-accent/60`.
- Группа (аккордеон) внутри блока: сохраняем, но шеврон 12px, отступ дочерних `ml-[26px]` без вертикальной линии (убираем `border-l`, уменьшаем визуальный шум).
- Бейдж-счётчик: переезжает в правый край, `min-w-[18px] h-[18px]`, `rounded-full bg-destructive/90`.
- Role-бейдж сверху делаем компактнее: одна строка `Роль · ФИО позиция` в `text-[11px]`.
- Анимация раскрытия группы — `transition-[max-height] duration-200`.

## 4. Технические изменения

Файлы:
- `src/components/AppSidebar.tsx` — полная переработка структуры `getNavItems()`: возвращает `NavSection[]`, где `NavSection = { key, label, entries: NavEntry[] }`. Рендер: map по секциям → заголовок + entries.
- Новый тип:
  ```ts
  interface NavSection { key: 'comm'|'analytics'|'hr'|'motivation'|'knowledge'|'system'; label: string; entries: NavEntry[] }
  ```
- Пустые секции (нет entries для роли) — не рендерятся.
- `src/i18n/locales/ru.json` (и других локалей) — добавить ключи `nav.sections.communication`, `nav.sections.analytics`, `nav.sections.hr`, `nav.sections.motivation`, `nav.sections.knowledge`, `nav.sections.system`.
- `src/index.css` — при необходимости токен `--sidebar-section-label` (или используем существующий `--sidebar-foreground` с opacity).
- `AppLayout.tsx` — обновить ширины 72→64, 260→240, если они захардкожены.

Бизнес-логика, роуты, RBAC — не трогаем. Это чистый presentation-рефакторинг.

## 5. Что НЕ делаем в этом этапе

- Не вводим двух-уровневый rail (как Meta) — это отдельная большая работа.
- Не меняем иконки на собственный пак.
- Не трогаем мобильный drawer кроме того, что он автоматически получит ту же сетку.
- Не делаем поиск по меню (можно следующим этапом, если попросите).

## 6. Порядок реализации (после одобрения)

1. Обновить i18n-ключи названий секций.
2. Переписать `getNavItems()` → `getNavSections()` с распределением per-role.
3. Обновить разметку рендера (секции + новый стиль активного пункта/бара/заголовков).
4. Подогнать `AppLayout` ширины.
5. Прогон `tsgo`, визуальная проверка для всех 5 ролей через preview.
