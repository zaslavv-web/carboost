## Что сделать на сервере

Вы сейчас находитесь в каталоге `backend`, поэтому команды фронта там не сработают. Плюс на сервере не установлен `bun`, значит используем `npm`, как уже предусмотрено в проекте.

### Команды

```bash
# 1. Перейти в корень проекта, НЕ в backend
cd /home/gro7659365/growth-peak.pro/docs

# 2. Обновить код
# если git pull уже делали — можно пропустить
git pull

# 3. Установить frontend-зависимости через npm
npm install


[gro7659365@gro7659365 docs]$ npm install
-bash: npm: command not found
[gro7659365@gro7659365 docs]$


# 4. Собрать frontend
npm run build
```

### Если сайт раздаёт `dist` напрямую

После успешной сборки обычно больше ничего не нужно: Vite перезапишет папку `dist/`, и nginx начнёт отдавать новую версию.

### Если сайт запущен через Docker/контейнер

Тогда после сборки нужно перезапустить web-контейнер, например:

```bash
docker compose restart web
```

### Проверка

После сборки открыть прод и проверить под HRD:

- появляется выбор режима при первом входе;
- доступен Today / daily work;
- из классического сайдбара есть возврат в Today.
