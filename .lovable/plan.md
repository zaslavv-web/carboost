## A. Лендинг (главная)

### A1. Авторская инфографика модулей (экран «16 инструментов»)

- Вместо `lucide`-иконок в `ModulesGrouped` / `ModuleTile` рисуем 16 собственных мини-композиций в inline SVG (по одной на модуль: Оргструктура, Карьерные треки, OKR, 1:1, Обучение, Геймификация, Опросы, Риски, Найм, Адаптация, Performance, Магазин, Признание, Аналитика, Трекер задач, AI-ассистент).
- Каждая инфографика — 2–4 слоя (фигура + акцентный элемент + «трек движения»), стиль единый: тонкие линии 1.5px, антик-голд акцент, плашка-подложка из card.
- Анимация «от старта к пику»: общий приём — точка/линия/столбик движется от начала к пику (рост столбца, заполнение круга, движение по треку, восхождение к звезде). Реализую через CSS keyframes + `stroke-dasharray`/`transform`, запуск по `IntersectionObserver` (`useInView` уже есть), на hover — replay.
- Размер тайлов увеличиваем (icon: текущие ~50% → 64–72% площади тайла; min-height тайла +20%). Сетка/верстка `ModulesGrouped` остаётся той же — 4 группы × 4 модуля.
- Новые файлы: `src/components/landing/infographics/<slug>.tsx` (16 шт.) + `src/components/landing/infographics/index.tsx` (реестр slug→компонент) + общие классы анимаций в `index.css` (`@keyframes climb`, `draw-line`, `fill-up`, `pulse-peak`).

## B. Продукт — критические баги

### B1. Белый экран на мобиле (продукт перестал открываться)

Гипотезы по приоритету:

1. `WorkspaceShell` имеет `h-[calc(100vh-3rem)]` и `-m-3 md:-m-8` — на мобиле перекрывает `AppLayout`, плюс `MobileEmployeeLayout` мог сломаться после правок sidebar.
2. Ошибка рантайма в одном из новых компонентов (`ContextRail`, `InspectorStack`, `useDockPanel` с `localStorage` при SSR-подобной гидрации) роняет всё дерево, т.к. нет ErrorBoundary на уровне роутов.
3. `MobileEmployeeLayout` ссылается на удалённые/переименованные пункты меню.

Действия:

- Воспроизвожу через Playwright (mobile viewport 390×844), снимаю console + screenshot.
- Оборачиваю `RoleAwareLayout` в `ErrorBoundary` с понятным фолбэком.
- Чиню первопричину по логам.

### B2. Дубликаты страниц

- `Dashboard` (employee) vs `HRDDashboard` vs `Employees` vs `UsersManagement`:
  - **Сотрудники (`/employees` → `HRDDashboard`)** оставляем как HR-витрину: карта, риски, аналитика по людям.
  - **Пользователи (`/users` → `UsersManagement`)** — оставляем только админ-функции: приглашения, роли, активация, удаление.
  - Удаляю пересекающиеся таблицы/виджеты из `HRDDashboard`, делаю ссылку «Управление пользователями →».
- **Дашборд HRD vs «Сотрудники»**: `RoleDashboard` для HRD сейчас рендерит `HRDDashboard` — то же, что `/employees`. Делаю отдельный `HRDOverview` (KPI + быстрые действия + последние события), а `/employees` оставляю списком.

### B3. Карьерные треки не отображаются

- Проверяю `CareerTrack.tsx` и `CareerTracksManagement.tsx`, эндпоинты `laravelDb` для `career_tracks`. Скорее всего сломался запрос после миграций трекера. Чиню запрос/политику + добавляю состояние загрузки/ошибки вместо пустого экрана.

### B4. Risk Analytics — некликабельные хайлайты

- В `RiskAnalytics.tsx` делаю карточки хайлайтов кликабельными → переход к отфильтрованному списку сотрудников (`/employees?risk=high`) или открытие drawer'а с деталями.

### B5. HRD видит всех пользователей всех компаний

- Серьёзный баг мультитенантности. Проверяю:
  - `CompanyScope` применён ко всем моделям, выдающим список (User, Profile, EmployeeRisk, …).
  - API-эндпоинт списка пользователей для HRD не использует «superadmin bypass» по ошибке.
  - Возможно, эндпоинт идёт мимо моделей (raw query) — там добавляю явный `where company_id`.
- Добавляю Feature-тест: HRD компании A не видит пользователей компании B.

### B6. Performance Reviews — «куски кода» в UI

- В `Performance.tsx` где-то рендерится не отформатированный JSON/raw markdown. Нахожу и оборачиваю в `react-markdown` или `JSON.stringify(…, null, 2)` внутри `<pre>` / нормальную карточку.

### B7. Полный аудит страниц

- Прохожу Playwright-скриптом по всем роутам из `App.tsx` под 3 ролями (employee/manager/hrd) + superadmin, снимаю скриншоты + console errors, чиню всё, где есть runtime error, белый экран или явный визуальный мусор. Результат — таблица «роут → статус → правка».

## Технические детали

- Инфографика: **inline SVG** (не PNG/Lottie) — чтобы темы/цвета жили через `currentColor` и токены, и анимация — CSS-only.
- Анимации: общий хук `useInView` уже есть, добавлю variant с `replay` по hover.
- Видео-хостинг для A5: предпочитаю Cloudinary (бесплатный план, прямой `.mp4` URL, без вотермарки на free до 25 кредитов/мес). Альтернатива — Bunny Stream demo или GitHub Release asset (CDN jsdelivr).
- Мультитенантность: усиление через тест + ревью всех `Api/*Controller` методов `index()` по списку моделей с `company_id`.
- ErrorBoundary: один компонент на уровне `RoleAwareLayout` + лог в консоль.
